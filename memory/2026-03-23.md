## Lessons Learned

- **Always test migration scripts against current file format, not assumed format.** The Python script had been silently failing (0 inserts) since the signals.md format changed. Idempotent upsert logic masked the failure — no errors, just no data.
- **Signal Scout runs clean as a manual trigger.** Use `openclaw cron run <id>` for on-demand debug runs.
- **jenny100x consensus is unusually strong this month.** All 3 models independently prioritized proof/case study as #1. That level of agreement is rare and actionable.
- **activity_log (hypertable) vs signals (table) are separate stores.** Don't conflate them. Python migration → activity_log. JS migration → signals. Both valid.
